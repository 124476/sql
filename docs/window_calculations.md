# Вычисления по окнам

## ROW_NUMBER

присваивает каждой строке уникальный порядковый номер внутри заданного окна

```
SELECT
    value_date,
    value,
    ROW_NUMBER() OVER (ORDER BY value DESC) AS rank_by_value
FROM table;
```

| Id | Значение | Номер |
|----|----------|-------| 
| 1  | 1000     | 1     |
| 2  | 800      | 2     |
| 3  | 800      | 3     |
| 4  | 200      | 4     |

## RANK

Если ROW_NUMBER всегда разные номера присваивает,
то RANK у равных по значениям позициям присваивает равные номера

| Id | Значение | Номер |
|----|----------|-------| 
| 1  | 1000     | 1     |
| 2  | 800      | 2     |
| 3  | 800      | 2     |
| 4  | 200      | 4     |

## DENSE_RANK

Он в отличие от RANK не пропускает номера по росту, а идет последовательно

| Id | Значение | Номер |
|----|----------|-------| 
| 1  | 1000     | 1     |
| 2  | 800      | 2     |
| 3  | 800      | 2     |
| 4  | 200      | 3     |


## LAG

доступ к предыдущей строке

```
SELECT id, amount,
    LAG(amount) OVER (
        ORDER BY id
    ) AS previous_amount
FROM table;    
```

| id | amount | previous_amount |
|----|--------|-----------------| 
| 1  | 1000   | NULL            |
| 2  | 800    | 1000            |
| 3  | 800    | 800             |
| 4  | 200    | 800             |

## LEAD

доступ к следующей строке

```
SELECT id, amount,
    LEAD(amount) OVER (
        ORDER BY id
    ) AS previous_amount
FROM table;    
```

| id | amount | previous_amount |
|----|--------|-----------------| 
| 1  | 1000   | 800             |
| 2  | 800    | 800             |
| 3  | 800    | 200             |
| 4  | 200    | NULL            |

> [!NOTE]
> Для обращение к заследующим строкам надо писать LAG(column, 2) и LEAD(column, 2)
